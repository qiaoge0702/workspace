# 功能开发检查清单

> 行为铁律见 SOUL.md。本清单只列开发专属检查项。

---

## 执行前

- [ ] **复述需求**，确认理解正确
- [ ] **明确边界**，确认不涉及哪些模块
- [ ] **搜索现有代码**: `grep -r "关键词" src/`
- [ ] **查阅设计文档**和 SNAPSHOT
- [ ] **确认架构位置**: 放哪个模块？符合分层？需要新建文件？

## 执行中

- [ ] **遵循项目开发规范**（命名、分层、字段格式）
- [ ] **复杂函数有文档字符串**
- [ ] **无重复代码**（>50行）
- [ ] **无死代码/未使用导入**

## 执行后

- [ ] **测试功能正常**
- [ ] **测试向后兼容**
- [ ] **检查循环依赖**
- [ ] **更新 CHANGELOG**
- [ ] **更新 MEMORY.md**（关键决策）
- [ ] **更新相关架构文档**

## 禁止行为 ❌

- [ ] 复制粘贴 >100行代码
- [ ] 新建文件不更新架构文档
- [ ] 不读现有代码就写新代码
- [ ] 绕过现有抽象层直接实现
