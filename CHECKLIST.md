# 功能开发检查清单

## 执行前（必须）

### 1. 精准理解需求
- [ ] 复述需求，确认理解是否正确
- [ ] 明确功能边界，确认不涉及哪些模块
- [ ] 确认优先级和交付时间

### 2. 现有代码搜索
- [ ] 搜索功能关键词: `grep -r "功能关键词" src/`
- [ ] 查看相关模块的 `__init__.py` 导出列表
- [ ] 阅读相关设计文档和 SNAPSHOT
- [ ] 检查是否有相关重构计划

### 3. 功能边界确认
- [ ] 这是新功能还是优化现有功能？
- [ ] 现有功能能否通过扩展满足需求？
- [ ] 如果不能，是否需要先重构？

### 4. 架构位置确认
- [ ] 功能应该放在哪个模块？
- [ ] 是否符合现有架构分层？
- [ ] 是否需要新建文件？（充分理由）

## 执行中（严守）

### 5. 严禁蔓延
- [ ] 不修改未涉及的文件
- [ ] 不添加未要求的功能
- [ ] 不跨模块处理（除非明确授权）
- [ ] 发现关联问题 → 记录并汇报，不擅自修复

### 6. 代码规范
- [ ] 遵循项目开发规范
- [ ] 复杂函数有文档字符串
- [ ] 无重复代码（>50行）
- [ ] 无死代码/未使用导入

## 执行后（必做）

### 7. 验证
- [ ] 测试功能正常
- [ ] 测试向后兼容
- [ ] 检查是否引入循环依赖

### 8. 文档更新
- [ ] 更新 CHANGELOG
- [ ] 更新 MEMORY.md（关键决策）
- [ ] 更新相关架构文档（如需要）

## 禁止行为 ❌

- [ ] 禁止复制粘贴 >100行代码
- [ ] 禁止新建文件而不更新架构文档
- [ ] 禁止不读现有代码就写新代码
- [ ] 禁止绕过现有抽象层直接实现
- [ ] 禁止猜测意图、擅自蔓延
